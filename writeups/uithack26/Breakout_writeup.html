1. Challenge Statement and Initial Idea

The prompt was essentially: Can you break out of this? follow the hint and find the flag. That wording strongly suggests that the visible game itself is not the real endpoint. Instead, the game likely contains some secondary path, hidden asset, or alternate code path that needs to be reached by analysis rather than by normal play.

The title breakout also hints at two separate meanings:

  • Literal meaning: it is a Breakout-style game.
  • Reversing meaning: we may need to “break out” of the normal game flow.

That immediately made the plan fairly clear:

  1. Identify the binary format and protections.
  2. Inspect how the game stores levels.
  3. Look for any hidden level handling or out-of-range behavior.
  4. Reconstruct anything unusual from static data if dynamic execution is unnecessary.

3. Following the Hint in the Visible Levels

The prompt explicitly says to follow the hint. In a Breakout game, the most obvious place for a hint is the brick layout itself. So the next step was to inspect how levels are stored.

The normal level loader uses a static table of level maps in the executable’s data section. Each level entry consists of an 8-byte header followed by a 13 x 19 tile map of 32-bit values. Dumping those maps reveals that one of the layouts is not decorative noise at all: it spells text. 

from pathlib import Path
import struct

p = Path('/home/userhonest/uithack26/breakout/breakout/breakout.exe').read_bytes()
base = 0xd2600

for lvl in range(10):
    off = base + lvl * 0x3e4
    hdr = struct.unpack_from('<II', p, off)
    vals = struct.unpack_from('<' + 'I' * (13 * 19), p, off + 8)
    print('LEVEL', lvl, 'hdr', hdr)
    for r in range(13):
        row = vals[r * 19:(r + 1) * 19]
        print(''.join('█' if x else ' ' for x in row))
    print()

The important discovery is that the visible layout spells BREAK and OUT. That is not the flag. It is a hint telling us what to do conceptually: move past the normal game boundaries.

This is the point where a lot of solvers can waste time trying to “play better.” But the binary itself is suggesting that the real content lies outside the standard visible levels.

4. Finding the Alternate Level Path

Once the normal level handling is understood, the next question is simple: what happens when the level index goes out of range?

The normal loader handles levels 0..9. That already suggests a branch somewhere that decides: use regular table lookup if the level number is valid, otherwise do something else. This “something else” turned out to be the entire challenge.

In the disassembly, the regular level loading path eventually branches to a different function when the level number exceeds the normal range. Instead of pulling a 13 x 19 board from the standard level array, it synthesizes a new board from a compact bitmask-like table stored in .rdata.

Component Meaning
Normal levels Stored directly in the data section as explicit tile maps.
Out-of-range level path Generates a board procedurally from an encoded table.
Encoded source table Begins at .rdata RVA 0x1400da0c0.
Hidden board dimensions 222 x 7

That board size is a giant clue. A 222 x 7 bitmap is not a normal Breakout level. It is exactly the sort of dimensions you would expect for horizontally encoded 5x7 or 7-pixel-tall text.

5. Extracting the Hidden Encoded Data

The hidden board is generated from bytes in .rdata. To inspect that table directly, we can convert the RVA into a file offset using the PE section layout. In this case, the raw file offset is: 

raw_offset = 0xd8000 + (0x1400da0c0 - 0x1400da000)

Then the raw bytes can be dumped very simply: 

from pathlib import Path

p = Path('/home/userhonest/uithack26/breakout/breakout/breakout.exe').read_bytes()
off = 0xd8000 + (0x1400da0c0 - 0x1400da000)
print(p[off:off + 256].hex())

Dumping those bytes by itself does not reveal readable ASCII. That is expected. The board is not stored as plain text. It is reconstructed bit by bit by the alternate loader.

6. Reconstructing the Hidden Board

The hidden loader computes a bitmap by iterating through rows and columns and deriving each cell from an encoded byte stream. The important recovered logic is: 

rows, cols = 222, 7

for row in range(rows):
    idx = row
    for col in range(cols):
        a = idx >> 3
        val = ((0x5a + 13 * a) & 0xff) ^ table[a]
        bit = (val >> (idx & 7)) & 1
        idx += 0xde   # 0xde == 222

There are several things happening here:

  • The board is conceptually indexed linearly.
  • a = idx >> 3 selects which encoded byte contains the relevant bit.
  • ((0x5a + 13 * a) & 0xff) is a position-dependent mask.
  • The byte from the table is XORed with that mask.
  • The specific bit inside the resulting byte is then extracted with (idx & 7).

So this is not encryption in any serious sense. It is just an obfuscated bitmap encoding.

Full Reconstruction Script

The following script reconstructs the hidden board from the executable, rotates it into human-readable orientation, and prints an ASCII preview. This is the essential solve logic. 

from pathlib import Path

exe = Path('/home/userhonest/uithack26/breakout/breakout/breakout.exe').read_bytes()

# Hidden data table in .rdata
table_off = 0xd8000 + (0x1400da0c0 - 0x1400da000)
table = exe[table_off:table_off + 256]

rows, cols = 222, 7

# Reconstruct hidden board as generated by the alternate level loader
mat = [[0] * cols for _ in range(rows)]

for row in range(rows):
    idx = row
    for col in range(cols):
        a = idx >> 3
        val = ((0x5a + 13 * a) & 0xff) ^ table[a]
        bit = (val >> (idx & 7)) & 1
        mat[row][col] = bit
        idx += 0xde

# Rotate to read as text: 7 rows high, 222 columns wide
rot = [[mat[c][x] for c in range(rows)] for x in range(7)]

for y in range(7):
    print(''.join('#' if bit else '.' for bit in rot[y]))

Once rotated, the output is clearly a text banner rather than a game board. The bitmap is effectively a custom 5x7-ish font rendered horizontally across a very wide strip.

7. Why the First Flag Guess Was Wrong

When a bitmap banner like this is viewed quickly, it is tempting to read it with approximate OCR or visual guessing. The first rough reading looked like:

The user correctly pointed out that this was wrong. That meant at least one glyph had been misread.

The correct fix was to stop guessing from the whole banner and instead segment it into individual glyphs, then compare the 5x7 bitmaps directly.

8. Full Solve Script With Decoding and Banner Segmentation

Below is a more complete version of the solve script. It reconstructs the hidden board, rotates it, segments the text by blank columns, and prints a simple textual view of each character. This is the kind of script you can keep as a reproducible solution rather than relying on manual reading. 

from pathlib import Path

EXE_PATH = '/home/userhonest/uithack26/breakout/breakout/breakout.exe'

def load_hidden_table():
    data = Path(EXE_PATH).read_bytes()
    off = 0xd8000 + (0x1400da0c0 - 0x1400da000)
    return data[off:off + 256]

def reconstruct_hidden_board(table):
    rows, cols = 222, 7
    mat = [[0] * cols for _ in range(rows)]
    for row in range(rows):
        idx = row
        for col in range(cols):
            a = idx >> 3
            val = ((0x5a + 13 * a) & 0xff) ^ table[a]
            bit = (val >> (idx & 7)) & 1
            mat[row][col] = bit
            idx += 0xde
    return mat

def rotate_board(mat):
    rows = len(mat)
    cols = len(mat[0])
    return [[mat[c][x] for c in range(rows)] for x in range(cols)]

def column_is_blank(rot, x):
    return all(rot[y][x] == 0 for y in range(len(rot)))

def segment_glyphs(rot):
    width = len(rot[0])
    glyphs = []
    x = 0
    while x < width:
        while x < width and column_is_blank(rot, x):
            x += 1
        if x >= width:
            break
        start = x
        while x < width and not column_is_blank(rot, x):
            x += 1
        end = x
        glyph = []
        for y in range(len(rot)):
            glyph.append(rot[y][start:end])
        glyphs.append(glyph)
    return glyphs

def render_glyph(g):
    return '\n'.join(''.join('#' if cell else '.' for cell in row) for row in g)

def main():
    table = load_hidden_table()
    mat = reconstruct_hidden_board(table)
    rot = rotate_board(mat)

    print('Hidden banner:')
    for row in rot:
        print(''.join('#' if bit else '.' for bit in row))

    print()
    print('Segmented glyphs:')
    for i, glyph in enumerate(segment_glyphs(rot)):
        print(f'glyph {i}')
        print(render_glyph(glyph))
        print()

if __name__ == '__main__':
    main()

If you run a script like this and compare repeated glyphs, the corrected reading becomes much more robust than a one-pass visual guess.

9. Challenge Logic Summary

The core idea of the challenge is elegant:

  1. The visible game provides a hint through its brick layout: BREAK OUT.
  2. Normal gameplay content lives in a straightforward level table.
  3. The real secret is not in those visible levels, but in the alternate loader for out-of-range level indices.
  4. That loader reconstructs a wide hidden bitmap from obfuscated bytes.
  5. The bitmap, when rotated and read correctly, contains the flag.

This is a nice reversing challenge because it rewards reading the binary’s real control flow instead of obsessing over superficial gameplay. The “follow the hint” instruction is literal and fair: the game says BREAK OUT, and the solve is to break out of the normal level system.
