Initial Triage

The directory contained a single ELF binary named pkzip_scrambler. Running it without arguments showed a usage banner: 

==================================
  PKZip Scrambler v2.04
  (c) 1994 PKWare Inc.
==================================
Usage: pkzip_scrambler <reg-key>

Basic inspection showed that the outer binary was stripped and dynamically linked. Strings also revealed references to memfd_create, /proc/self/fd/%d, and an error message about an invalid ELF payload. That strongly suggested a small packer or loader.

Outer Binary Analysis

Disassembling the outer binary showed a straightforward flow in main:

  • Allocate a large buffer.
  • XOR-decode a blob from .rodata using repeating bytes from 0xDEADBEEF.
  • Run a custom decompression routine on the decoded data.
  • Verify that the output starts with 0x7F 45 4C 46 (ELF).
  • Write the unpacked payload into a memfd and execv it, forwarding the supplied registration key.

At that point, the real license check was clearly in the embedded second-stage ELF rather than the outer loader.

Unpacking the Payload

I reproduced the loader logic in a short script: 

# 1. Read encoded blob from the host binary
# 2. XOR each byte with the repeating key bytes from 0xDEADBEEF
# 3. Run the custom LZ-style decompressor
# 4. Write the result as payload.elf

The output was an unstripped, statically linked ELF. That changed the problem from packed reverse engineering into a simple symbol-guided inspection job.

Second-Stage License Check

The unpacked payload’s main contained the actual validation logic. It required:

  • Exactly 16 input bytes.
  • Interpretation as four 32-bit little-endian words A, B, C, D.
  • Validation against four arithmetic and XOR constraints.
A ^ B == 0x08282820
B + C == 0xD3A5AED3
C ^ D == 0x08075242
3*A + D == 0xB6437B11

Solving these constraints with printable ASCII byte restrictions produced multiple valid keys. One of them was: 

PkZipCrack3r!94z

Getting the Flag

Running the original wrapper binary with that key returned the success message and printed the flag: 

$ ./pkzip_scrambler 'PkZipCrack3r!94z'
Registration successful! Welcome, licensed user.
Flag: UiTHack26{y0u_unp4ck3d_th3_pkz1p_scr4mbl3r}
The important observation was that the visible binary was only a loader. Once the embedded ELF was unpacked, the license validation reduced to a small set of integer equations.
