UiTHack26 Crypto Writeup

Half Baked Writeup

We are given a custom password hashing program, a leaked password file, a decryptor, and an encrypted movie script. The intended break is that the hash only preserves the low nibble of each of the first eight password characters, which makes the stored value reversible enough to recover a working password for the decryptor.

Target account
sysadmin / P. Hansen
Bug
Only 4 bits per character survive
Exploit
Recovered password material decrypts the file

Initial Recon

  • crypt.pas implements the custom password hashing routine.
  • passwd.bak contains stored password hashes.
  • decrypt is the binary used to unlock the movie file.
  • movie.encrypted is the ciphertext we need to recover.

The account that matters is easy to spot in the backup:

sysadmin:lsf306ed84e8bfd4c1:42:42:P.Hansen:/home/sysadmin:/bin/sh

The hash field is lsf306ed84e8bfd4c1, so the two-character salt is ls and the eight encoded bytes are f3 06 ed 84 e8 bf d4 c1.

Finding The Bug

The important bug is in Transform. Every password character is reduced to its low nibble before any mixing happens. 

function Transform(c: char): byte;
var
  v : byte;
begin
  v := ord(c);
  v := v AND (v XOR (v AND $F0));
  Transform := v AND $0F;
end;

This destroys most of the entropy immediately. The code also uses only the first MAX_PW = 8 characters, so the value that actually gets hashed is just eight 4-bit nibbles. The rest of the algorithm is only XOR, byte addition, and a small permutation, so reversing it is practical.

So the custom hash is not acting like a one-way function here. It is closer to a reversible encoding of eight nibbles plus some salt-dependent scrambling.

Reversing The Hash

Starting from the stored bytes f3 06 ed 84 e8 bf d4 c1, we reverse the pipeline in the opposite order: undo the final nibble-XOR pass, undo the permutation in Scatter, then undo the salt-dependent recurrence in Diffuse.

1. Undo The Final Pass

The last loop in Scatter XORs each byte with the low nibble of the next byte. Since each byte only depends on the byte to its right, it can be inverted by walking backward: 

for i = 6 down to 0:
    key[i] = key[i] XOR (key[i + 1] AND 0x0f)

This gives the state before the final pass:

fe 0d eb 86 e2 ba d5 c1

2. Undo The Swap/XOR Mixing

The first part of Scatter only touches positions 0..2 and their mirrored bytes, while indices 3 and 4 stay visible. That makes the permutation easy to solve directly. Reversing it produces the state leaving Diffuse: 

23 37 58 86 e2 6d 55 c9

3. Undo The Salt Mixing

The salt alphabet is: 

./0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz

For ls, the character positions are 49 and 56. That gives: 

saltval = (56 << 6) | 49 = 3633
slo = 0x31
shi = 0x0e

Undoing Diffuse yields the original packed block: 

12 39 04 14 00 00 00 00

The first four bytes contain the eight recovered password nibbles: 

12 39 04 14  ->  1 2 3 9 0 4 1 4

A convenient representative password is: 

12390414

The original text password is not unique. Any eight-character string with the same low nibble pattern would produce the same hash.

Breaking The Decryptor

The decryptor uses the same reduced password material. For the representative password 12390414, the effective key bytes are: 

12 39 04 14 00 00 00 00

The file is then decrypted with a chained XOR keystream. In simplified form: 

prev = key[0]
for i in range(len(ciphertext)):
    t = prev ^ key[i % 8] ^ (i & 0xff)
    prev = t
    plaintext[i] = ciphertext[i] ^ t

Once the recovered password is used, movie.encrypted decrypts into a readable plaintext screenplay.

Solve Summary

  1. Read crypt.pas and notice that only the low nibble of each of the first eight characters matters.
  2. Extract Hansen's hash from passwd.bak: lsf306ed84e8bfd4c1.
  3. Reverse the hash to recover the nibble sequence 1 2 3 9 0 4 1 4.
  4. Build a representative password such as 12390414.
  5. Use the same reduced key material against the decryptor or reproduce its XOR loop.
  6. Search the decrypted script for the real flag string.

Reference Script

The script below performs the full solve locally: reverse the hash, derive a representative password, decrypt the movie file, and extract the flag. 

const fs = require('fs');

const salt = 'ls';
const hex = 'f306ed84e8bfd4c1';
const SALT_CHARS = './0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz';

function reverseHash(salt, hex) {
  let key = Array.from({ length: 8 }, (_, i) =>
    parseInt(hex.slice(i * 2, i * 2 + 2), 16)
  );

  for (let i = 6; i >= 0; i--) {
    key[i] ^= (key[i + 1] & 0x0f);
  }

  let d = new Array(8);
  d[3] = key[3];
  d[4] = key[4];
  d[0] = key[7] ^ d[4];
  d[1] = key[6] ^ d[4];
  d[2] = key[5] ^ d[4];
  d[7] = key[0] ^ d[1];
  d[6] = key[1] ^ d[2];
  d[5] = key[2] ^ d[3];

  const pos1 = SALT_CHARS.indexOf(salt[0]);
  const pos2 = SALT_CHARS.indexOf(salt[1]);
  const saltval = (pos2 << 6) | pos1;
  const slo = saltval & 0xff;
  const shi = (saltval >> 8) & 0x0f;

  let packed = new Array(8);
  packed[0] = d[0] ^ slo;
  packed[1] = d[1] ^ shi;
  for (let i = 2; i < 8; i++) {
    packed[i] = d[i] ^ ((d[i - 2] + d[i - 1] + i) & 0xff);
  }

  let nibbles = [];
  for (let i = 0; i < 4; i++) {
    nibbles.push((packed[i] >> 4) & 0x0f, packed[i] & 0x0f);
  }

  return { packed, nibbles };
}

function representativePassword(nibbles) {
  return nibbles.map(n => n.toString(16)).join('');
}

function decryptFile(path, packed) {
  const ciphertext = fs.readFileSync(path);
  let prev = packed[0];
  let plaintext = Buffer.alloc(ciphertext.length);

  for (let i = 0; i < ciphertext.length; i++) {
    const t = (prev ^ packed[i % 8] ^ (i & 0xff)) & 0xff;
    prev = t;
    plaintext[i] = ciphertext[i] ^ t;
  }

  return plaintext;
}

const { packed, nibbles } = reverseHash(salt, hex);
const password = representativePassword(nibbles);
const plaintext = decryptFile('half_baked/challenge/movie.encrypted', packed);
const text = plaintext.toString('utf8');
const realFlag = (text.match(/UiTHack26\{[^}\n]+\}/) || [])[0];

console.log('nibbles:', nibbles.join(' '));
console.log('password:', password);
console.log('packed:', packed.map(x => x.toString(16).padStart(2, '0')).join(' '));
console.log('flag:', realFlag);

Flag Extraction

After decryption, the movie script contains several brace-delimited strings. The real flag is the one with the challenge prefix:

UiTHack26{3t_n1bbl3_h0m3_y0u_0nly_n33d_th3_f1rst_31ght}

Takeaway

The whole challenge hinges on one bad design choice: only the low nibble of the first eight password characters is preserved. That collapses the password space, makes the stored hash reversible enough to recover working password material, and breaks the decryptor as soon as the same secret is reused there.

Final Answer
